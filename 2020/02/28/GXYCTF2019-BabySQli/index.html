前言：

 蛮久没刷题了，颓废了一段日子，现在开始冲冲冲呀！

一个新知识点

 当查询的数据不存在的时候，联合查询就会构造一个虚拟的数据。
 例如，数据表里有3个字段，我使用select * from _sql where id =111没有返回任何数据(表里最大id为10)。那么我们尝试使用union select 联合查询返回数据。使用select * from _sql where id =111 union select 1,2,3;(前题是已经知道表里有几个字段)

 这样我们就可以自定义它的字段了。

开始解题

  1. 打开题目尝试万能密码无果。查看源代码，发现一个base32，解码后发现是base64 再解码 得到select * from user where username = '$name' 解码网址
  2. 利用前面的知识点使用union select 来构造。我们先来判断有几个字段使用name=1'union select 1,2# 报语法错误。name=1'union select 1,2,3# 报用户不存在，说明是3个字段。下面猜测admin在哪个字段，我们知道有admin用户的，经测试name=1'union select 1,'admin',3#报密码错误，说明这里就是username了。
  3. 根据猜测password字段应该是3的位置，我们使用name=1'union select 1,'admin',3#&pw=3密码错误。那么还有一种可能就是我们登入时的password会MD5。那我们将123 MD5 后放在第3个字段，然后登入密码使用123。name=1'union select 1,'admin','202cb962ac59075b964b07152d234b70'#&pw=123 得到我们的flag大功告成！！！
