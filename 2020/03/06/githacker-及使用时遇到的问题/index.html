githacker 及使用时遇到的问题

发表于2020-03-06|更新于2020-03-06|推荐
|浏览量:

前言:

  在做 CTFhub Git泄漏之log时遇到了一些问题这里记一下。

githacker 和 githack 有什么区别呢？

 其实以前我一直使用的是 githack 但是这题却力不从心，他们长得还有点像呢。githack只拉源码没有复原git整个记录。
 这里用网上的一段话解析一下吧 :
GitHacker是一个多线程工具，用于检测站点是否存在git源码泄漏，并能够将网站源代码下载到本地。值得一提的是，这个工具会将整个git repo恢复到本地，而不是像[githack]那样，只是简单的恢复到最新版本。如此一来，你就可以查看到开发人员的历史提交记录以及提交注释，以更好的掌握开发者的性格和心理，从而为进一步的代码审计奠定基础。
直接百度 githacker  它的github下载说明也有详细的介绍。

如何使用 githacker

 它的使用也非常简单 python Githacker.py url/.git/即可。

  1. 正确下载安装 git ，我就是因为git安装这里出了点问题害我找了好久。
  2. 给python 安装 requests 库 。pip install requests。
  3. 下载 Githacker ，注意不要直接去下载压缩文件。git clone https://github.com/wangyihang/GitHacker.git
  4. 进入 Githacker 文件夹 使用命令python GitHacker.py [Website]/.git/

注：

贴一下githacker的下载地址 【python3环境】

解题过程

  1. 使用python GitHacker.py http://challenge-9533751a727342f1.sandbox.ctfhub.com:10080/.git/ 将泄漏的文件都下载到本地。
  2. cd 到下载下来的文件夹内，使用git log因为题目就是 log 泄漏。
  3. 现在我们有2种方法拿到flag 第一是直接比较不同得出flag，二是恢复到 有flag的状态。
    • 比较得flag 使用 git diff 2d815970d 得到flag。
    • 使用还原，git reset --hard 2d815970d然后 cat 这个文件也可得到flag 。

遇到的问题

  git log 可以得到想要的数据但是使用git diff 2d815970d 时报错fatal: unable to read 5ff04b5f 。
&esmp;进过排查发现是 git安装的问题 有些依赖库没有安所以导致下载git泄漏文件时不完整。
正确安装 git

  1. 安装git：yum install git 。
  2. 查看yum源仓库Git信息：yum info git 。
  3. 安装依赖库：yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel
    yum install gcc-c++ perl-ExtUtils-MakeMaker
  4. 查看版本信息：git –version

注：最好使用git 2.2以上的版本，虽然这里 log 问题解决了但是还是会有一些奇怪的错误，工具简介也规定了git版本。
