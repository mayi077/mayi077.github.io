SVN 是个什么东西

 SVN（Subversion）是一个开放源代码的版本控制系统，与使用Git版本控制器类似。在服务器上布署代码时。如果是使用 svn checkout 功能来更新代码，而没有配置好目录访问权限，则会存在此漏洞。黑客利用此漏洞，可以下载整套网站的源代码。

svn 目录

 使用 svn checkout 后，项目目录会产生一个隐藏的 .svn文件夹。
 svn1.6及以前版本会在项目的每个文件夹都生成一个.svn 文件夹，里面包含了所有文件的备份。
 svn1.7 及以后版本只在项目根目录生成一个.svn文件夹，里面的pristime文件夹里包含了整个项目的所有文件备份。它里面有一堆 00到ff 的文件夹，每个文件夹里有若干个.svn-base 文件，文件命名规则是文件原名的哈希值，然后文件夹名称其实是文件的开头2位数。
  .svn/会有一个wc,db 二进制文件，里面包含项目里的所有代码及每个文件名及对应哈希后的值（知道对应的哈希值有助于找flag）。
以上说的是1.7及以上版本

利用 dvcs-ripper 工具

 它可以把存在漏洞的网站.svn信息全部下载到本地便于我们更好的利用。它还可以利用于git,hg,bzr,cvs的等的信息泄漏。

dvcs-ripper 的安装

 这个工具是用Perl语言写的，所以需要安装 perl 环境而且具体漏洞还需要安装一些具体模块。
在kali 上安装及使用

  • sudo apt-get install perl libio-socket-ssl-perl libdbd-sqlite3-perl libclass-dbi-perl libio-all-lwp-perl 安装所需要的环境。
  • 下载及解压软件【软件地址】
  • 进入解压后的使用命令perl rip-svn.pl -v -u http://www.example.com/.svn/ 即可以下载 .svn泄漏的所有文件。
  • 当然肯能不会成功，那久要具体情况具体分析了。

    安装工具时遇到的一些困难

     linux 还是比较复杂的，使用过程出现了缺失模块的问题，安装模块又出现安装不上的问题。总之要善于看报错信息要积累要实践。（有机会系统的学习就更好了。）
  1. kali 上使用报错 缺少 DBD::SQLite 这个模块。按网上教程使用perl -MCPAN -e 'install DBD::SQLite'安装这个模块，总是连接超时。开始不明白以为是自己配置的源问题于是就是了解及修改了一波，最后却发现只是当时请求比较慢多试了几次就好了。
  2. centos 中安装后也报错缺失LWP这个模块，尝试去安装却报错在 但是我手动去这文件找却又可以找到。后面发现是我安装的模块不对,要使用perl -MCPAN -e "install Bundle::LWP"来安装该模块。

CTFhub–SVN泄漏

  1. 扫描发现有.svn/目录确认是 .svn泄露。使用 dvcs-ripper 工具中的 rip-svn.pl 脚本进行 clone。
  2. 使用命令perl rip-svn.pl -v -u http://challenge-e6b3d24d85a17e13.sandbox.ctfhub.com:10080/.svn/ 来下载代码。
  3. 使用 ls -al 来查看是否有新增文件夹如.svn，进入文件夹先看看 wc.db 文件内是否有为flag的文件名。 cat wc.db | grep -a flag。
  4. 可以看到flag文件名对应的哈希值，到.svn/pristime/中去找（ 哈希值开头2位为文件夹名 ）。当然这是一般情况，特殊情况可以多找找其它文件夹内的文件。
  5. cat 文件即可看到flag。

常用的 yum

 基于RPM包管理，能够从指定的服务器（配置的源）自动下载RPM包并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软体包，无须繁琐地一次次下载、安装。yum提供了查找、安装、删除某一个、一组甚至全部软件包的命令，而且命令简洁而又好记。centos中可以使用kali不行。软件包管理器的前端工具 。yum换源要下载新的.rpm文件，然后用rpm进行安装。

常用的 apt-get

 apt-get（Advanced Package Tool），是一条linux命令，适用于deb包管理式的操作系统，主要用于自动从互联网的软件仓库（repositories）中搜索、安装、升级、卸载软件或操作系统。使用apt-get的主流Linux系统包括Debian和Ubuntu变异版本。kali 及相关Linux发行版可用。

grep 的进一步了解

 grep命令是一种强大的文本搜索工具，它能使用正则表达式搜索文本，并把匹配的行打印出来。用于查看文件中是否包含 flag 字样再好不过了。
常用参数 ：

  • -a 或 –text : 不忽略二进制的数据。不用这参数查看二进制文件会报错。
  • -A<显示行数> ： 除了显示符合范本样式的那一列之外，并显示该行之后的内容。
  • -b : 在显示符合样式的那一行之前，标示出该行第一个字符的编号。
  • -B<显示行数> ：除了显示符合样式的那一行之外，并显示该行之前的内容。
  • -l : 列出文件内容符合指定的样式的文件名称。
  • -i : 忽略字符大小写的差别。
  • -H : 在显示符合样式的那一行之前，表示该行所属的文件名称。
