PHP Session 的序列化和反序列化

应用场景

  1. 一般来说服务器是很少关机重启的，但是有的时候必须需要重启，而用户会话还在进行相应的操作，这时就需要使用序列化 来把这个 session 信息保存起来放在硬盘里，服务器重启后又重新加载。这样保证了用户信息不会丢失，实现永久化保存。
  2. 淘宝每年都会有定时抢购的活动，很多用户会提前登录等待，长时间不进行操作，一致保存在内存中，而到达指定时刻，几十万用户并发访问，就可能会有几十万个session，内存可能吃不消，这时就需要进行对象的活化、钝化，让其在闲置的时候离开内存，将信息保存至硬盘，等要用的时候，就重新加载进内存

PHP 反序列化与 Session

PHP session 序列化机制

 当session_start()被调用或者php.ini中session.auto_start为1时，PHP内部调用会话管理器，访问用户session被序列化以后，存储到指定目录（默认为/tmp）。

session 序列化及反序列化处理器

  PHP 内置了一些处理器用于对 session 数据进行序列化及反序列化，常见的有以下三种：

  • php 对应的存储格式为 键名 + | + 经过 serialize 序列化处理的值。
  • php_binary 对应的存储格式为 键名长度对应的ASCII字符+ 键名 + 经过serialize 序列化的值。
  • php_serialize (php>=5.5) 对应的存储格式为 经过 serialize 序列化的值

    与 session 相关的配置项

     session 配置项有很多可选的，这里我只学习我们利用需要用到的。
  1. session.auto_start 表示是否自动启用 session 相关机制( 如果改它需要重启服务器 )，如果自动启动可以不用 session_start() 来调用。(可以不用尝试在代码里改它了)
  2. session.save_path 表明了，这个序列化文件存放在哪。
  3. session.serialize_handler 定义用什么序列化/反序列化处理器

PHP Session 中序列化的危害

 PHP中的Session的实现是没有的问题，危害主要是由于程序员的Session使用不当而引起的。
如果在PHP在序列化存储的$_SESSION数据时使用的处理器和反序列化使用的处理器不一样，会导致数据无法正确第反序列化。通过精心构造的数据包，就可以绕过程序的验证或者是执行一些系统的方法。

githacker 和 githack 有什么区别呢？

 其实以前我一直使用的是 githack 但是这题却力不从心，他们长得还有点像呢。githack只拉源码没有复原git整个记录。
 这里用网上的一段话解析一下吧 :
GitHacker是一个多线程工具，用于检测站点是否存在git源码泄漏，并能够将网站源代码下载到本地。值得一提的是，这个工具会将整个git repo恢复到本地，而不是像[githack]那样，只是简单的恢复到最新版本。如此一来，你就可以查看到开发人员的历史提交记录以及提交注释，以更好的掌握开发者的性格和心理，从而为进一步的代码审计奠定基础。
直接百度 githacker  它的github下载说明也有详细的介绍。

 它的使用也非常简单 python Githacker.py url/.git/即可。

非正常退出，系统自动备份

  很多软件会有临时文件，在编辑文本时就会创建的文件，如果程序正常退出，临时文件自动删除，如果意外退出就会保留，我们也可以将这交换文件叫作备份文件。下面我们介绍2中常见的自动备份文件。

.bak 文件

 Bak格式的文件其实是由软件或系统自动创建的备份文件，比如平常使用的Word、PhotoShop等绘图、文字处理软件修改了原文件的内容后，保存了修改后的内容，那么修改前的内容会自动保存为后缀名为bak的备份文件。
 想要打开这类文件可以用产生这个备份文件的原软件了，比如我们经常会有 index.php.bak 这么个文件，它其实是index.php的备份文件，我们直接可以用记事本类似的软件打开。或者把.bak这个后缀删掉再尝试打开。
 一般来讲 .bak文件可以通过直接重命名把bak替换成该文件的格式就可以恢复此文件并且正常打开了，不需要其他软件辅助打开。

VIM 不正常退出产生的swp文件

 当你非正常关闭vim编辑器时（比如直接关闭终端或者电脑断电），会生成一个.swp文件，这个文件是一个临时交换文件，用来备份缓冲区中的内容。需要注意的是如果你并没有对文件进行修改，而只是读取文件，是不会产生.swp文件的。
 意外退出时，并不会覆盖旧的交换文件，而是会重新生成新的交换文件。而原来的文件中并不会有这次的修改，文件内容还是和打开时一样。
如：
 第一次产生的交换文件名为“.filename.txt.swp”；再次意外退出后，将会产生名为“.filename.txt.swo”的交换文件；而第三次产生的交换文件则为“.filename.txt.swn” 注意前面有个.噢！

常用的命令
  1. 可以使用 vim -r 查看单前目录的所有swp文件。
  2. 可以使vim -r filename 来打开到上次意外退出没有保存的修改内容。 这也是做题中最常用到的命令了
